<callout icon="📌" color="blue_bg">
**노션 자동화를 만들고 싶은데, Make를 배워야 할까요? 아니면 AI한테 코드를 짜달라고 하면 될까요?**

둘 다 "직접 코딩하지 않는다"는 공통점이 있지만, 결과물과 적합한 사용자가 완전히 다릅니다. 비용, 유연성, 난이도를 기준으로 솔직하게 비교합니다.

**이 글에서 배울 내용** ⏳ `읽기 6분`

- 구체적인 비교 (비용, 난이도, 유연성)
- 어떤 사람에게 어떤 방식이 맞는지
</callout>

# 같은 목표, 다른 방법

"노션 DB에 새 항목이 추가되면 Slack으로 알림을 보내고 싶다."

이 한 가지 목표를 달성하는 방법이 두 가지 있습니다.

**방법 A — 노코드 (Make, n8n)**
화면에서 "노션" 블록과 "Slack" 블록을 선으로 연결합니다. 계정을 로그인하고, 어떤 DB인지 선택하고, 어떤 메시지를 보낼지 설정하면 끝. 코드는 한 줄도 없습니다.

**방법 B — 바이브코딩 (Claude Code 등)**
AI에게 "노션 DB 변경을 감지해서 Slack으로 알림 보내는 서버리스 함수 만들어줘"라고 말합니다. AI가 코드를 생성하고, 배포까지 도와줍니다. 코드는 있지만 내가 쓴 건 아닙니다.

# 노코드 자동화 — 블록을 연결하는 방식

Make, n8n, Zapier 같은 도구는 **레고 블록**처럼 생각하면 됩니다.

"이 서비스에서 이런 이벤트가 발생하면" (트리거)
→ "이 서비스에 이런 동작을 실행해" (액션)

이 조합을 시각적으로 만들어서 실행합니다.

## 잘하는 것

**빠릅니다.** "노션 → Slack 알림"은 진짜로 10분이면 만듭니다. 계정 연결하고 필드 매핑하면 끝이에요.

**눈에 보입니다.** 자동화가 어떻게 흘러가는지 시각적으로 확인할 수 있고, 각 단계의 데이터도 클릭하면 볼 수 있습니다. 어디서 문제가 생겼는지 바로 알 수 있습니다.

**템플릿이 있습니다.** "노션 + Google Calendar 동기화" 같은 흔한 자동화는 커뮤니티가 만들어둔 템플릿을 복사해서 바로 쓸 수 있습니다.

## 아쉬운 것

**복잡해지면 괴로워집니다.** "데이터를 정제해서, 조건별로 분기하고, API 응답을 파싱해서 가공한 뒤, 다른 DB에 기록"하는 수준이 되면 블록이 꼬이기 시작합니다.

**비용이 사용량에 비례합니다.** Make 무료 플랜은 월 1,000번 실행, 유료는 월 $9부터입니다. 자동화가 복잡할수록 한 번의 워크플로에서 여러 "실행"이 발생해서 비용이 빠르게 올라갑니다.

**AI 연동이 어색합니다.** Claude API나 GPT API를 호출하려면 HTTP 모듈에서 직접 헤더를 설정하고, JSON을 구성하고, 응답을 파싱해야 합니다. 가능은 하지만, 이 시점에서 "노코드의 편리함"은 상당 부분 사라집니다.

# 바이브코딩 — AI에게 코드를 시키는 방식

Claude Code, Cursor, Windsurf 같은 도구를 사용합니다. 자연어로 원하는 걸 설명하면 AI가 코드를 생성합니다.

"바이브코딩"이라는 이름은 Andrej Karpathy가 만든 용어입니다. **분위기(vibe)만 잡아주면 AI가 코딩한다**는 의미입니다.

## 잘하는 것

**뭐든 만들 수 있습니다.** 코드이기 때문에 제한이 없습니다. 프롬프트 캐싱으로 API 비용을 90% 줄이거나, 120명의 학생 데이터를 병렬 처리하거나, 에러 발생 시 Slack으로 알림을 보내는 것까지 — 상상하는 건 다 구현 가능합니다.

**비용이 거의 없습니다.** Vercel 무료 플랜으로 서버리스 함수를 돌리면, 실행 비용은 0원입니다. API 호출 비용(Claude, 노션 등)만 발생하고, Make처럼 "실행 횟수" 과금이 없습니다.

**코드를 직접 쓰지 않습니다.** 이게 핵심입니다. "노션 DB 읽어서 Claude로 생기부 만들어줘"라고 하면 AI가 코드를 만들고, 타입 체크하고, 배포까지 합니다.

## 아쉬운 것

**개념은 이해해야 합니다.** 코드를 직접 안 쓴다고 해서 아무것도 몰라도 되는 건 아닙니다. "API가 뭔지", "환경변수가 뭔지", "배포가 뭔지" 정도는 알아야 AI와 대화가 됩니다.

**시각적 디버깅이 없습니다.** Make처럼 "여기서 문제야"를 그림으로 보여주지 않습니다. 에러 로그를 읽어야 합니다. (물론 AI한테 "이 에러 뭐야?"라고 물어볼 수 있지만요.)

**수정도 코드 수준입니다.** 자동화를 바꾸려면 코드를 수정하고 재배포해야 합니다. 블록을 드래그해서 바꾸는 것보다는 번거롭습니다. 다만 AI에게 "여기 이렇게 바꿔줘"라고 하면 되니까 과거의 코딩과는 다릅니다.

# 비교 표

| 항목 | Make / n8n | 바이브코딩 |
|------|------------|------------|
| 진입 난이도 | 낮음 | 중간 |
| 단순 자동화 속도 | 10분 | 30분~1시간 |
| 복잡 자동화 속도 | 느리고 괴로움 | 빠르고 자유로움 |
| 월 비용 | $9~$30+ | $0~$5 |
| AI API 연동 | 가능하지만 번거로움 | 자연스러움 |
| 디버깅 | 시각적 | 로그 기반 |
| 유연성 | 블록 범위 내 | 제한 없음 |

# 그래서 뭘 써야 하나

<callout icon="💡" color="green_bg">
**노코드가 맞는 경우**
- "A가 발생하면 B를 실행"하는 단순한 자동화
- 프로그래밍 개념이 익숙하지 않은 분
- 10분 안에 결과를 보고 싶은 분
- 이미 Make/n8n을 쓰고 있는 분
</callout>

<callout icon="💡" color="green_bg">
**바이브코딩이 맞는 경우**
- AI API를 활용한 데이터 가공이 핵심인 자동화
- 월 구독 비용을 최소화하고 싶은 분
- 세부 동작을 직접 통제하고 싶은 분
- API, 배포 같은 개념을 이해하거나 배울 의향이 있는 분
</callout>

# 양자택일이 아니다

실제로 저는 두 방식을 섞어서 씁니다.

"DB 변경 → Slack 알림"처럼 단순한 건 Make로 5분 만에 만들고, "학생 120명의 활동 데이터 → AI 가공 → 생기부 초안 자동 생성"처럼 복잡한 건 바이브코딩으로 서버리스 함수를 만듭니다.

도구에 맞춰 문제를 끼워 맞추지 마세요. **문제의 복잡도에 맞는 도구를 선택**하면 됩니다.

<callout icon="📝" color="blue_bg">
**시리즈 핵심 정리**

1편: 노션 워커스는 커스텀 에이전트에 코드를 붙이는 알파 기능이다
2편: 코드를 짤 수 있다면, 서버리스 함수가 크레딧과 안정성 면에서 유리하다
3편: 단순 자동화는 노코드, 복잡한 자동화는 바이브코딩이 적합하다
</callout>
