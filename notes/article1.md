# 노션 워커스란 무엇인가

노션 워커스는 2026년 초에 공개된 알파 기능입니다. 한 마디로 요약하면 **노션 AI 커스텀 에이전트에 코드를 붙이는 방법**입니다.

기존 커스텀 에이전트는 노션 내부 데이터를 읽고, 자연어로 응답하는 것까지가 한계였습니다. 외부 API를 호출하거나, 데이터를 가공하거나, 특정 형식으로 문서를 생성하는 건 불가능했습니다.

워커스는 이 벽을 깹니다. TypeScript로 함수를 작성하고, 노션 서버에 배포하면, 에이전트가 대화 중에 해당 함수를 호출할 수 있습니다.

# 에이전트와 워커의 역할 구조

구조를 이해하려면 **리모컨과 기계**로 비유하면 됩니다.

| 구분 | 역할 | 하는 일 |
|------|------|---------|
| 커스텀 에이전트 | 리모컨 | 사용자의 자연어를 해석하고, 적절한 워커 도구를 호출 |
| 워커 (tool) | 기계 | 실제 로직 실행 — API 호출, 데이터 처리, DB 읽기/쓰기 |

사용자가 에이전트에게 "생기부 작성해줘"라고 말하면, 에이전트는 워커에 등록된 도구의 스키마(파라미터 설명)를 읽고, 자연어를 파라미터로 변환해서 워커를 호출합니다. OpenAI의 function calling이나 Claude의 tool use와 같은 원리입니다.

워커 코드에서 정의한 `description`과 `schema`가 에이전트의 판단 근거가 되므로, 이 부분을 잘 쓰는 게 중요합니다.

# 실제 배포 과정

워커를 만들고 배포하는 과정은 다음과 같습니다.

## ntn CLI 설치와 프로젝트 생성

```bash
npm i -g ntn
ntn login
ntn workers new
```

`ntn` CLI로 로그인하고 프로젝트를 스캐폴딩합니다. 생성된 `src/index.ts`에 도구를 정의합니다.

## 도구(tool) 정의

```typescript
worker.tool("sayHello", {
  title: "인사하기",
  description: "이름을 받아 인사말을 반환합니다",
  schema: j.object({
    name: j.string().describe("인사할 이름"),
  }),
  execute: ({ name }) => `안녕하세요, ${name}님!`,
});
```

`schema`로 입력 파라미터를 정의하고, `execute`에 실행 로직을 작성합니다. 비동기 함수도 지원하므로 외부 API 호출, DB 쿼리 등이 가능합니다.

## 배포와 연결

```bash
ntn workers deploy
```

배포하면 노션 서버에 워커가 등록됩니다. 이후 노션에서 커스텀 에이전트 설정 → 도구 추가에서 배포된 워커를 선택하면 연결 완료입니다.

# 현재 제약사항 (알파 상태)

<callout icon="⚠️" color="orange_bg">
**노션 워커스는 2026년 3월 기준 "extreme pre-release alpha" 상태입니다.** 프로덕션 용도로 사용하기엔 이른 시점이며, breaking change가 수시로 발생할 수 있습니다.
</callout>

알아두어야 할 제약사항을 정리합니다.

## 플랫폼 제한

`ntn` CLI는 macOS와 Linux만 지원합니다. Windows에서는 WSL(Windows Subsystem for Linux)을 설치해야 사용할 수 있습니다. 이 한 가지만으로도 진입 장벽이 상당히 올라갑니다.

## 기능 가용성

| 기능 | 상태 |
|------|------|
| Tool (도구) | 사용 가능 |
| OAuth | 사용 가능 |
| Sync (동기화) | 사용 가능 (알파) |
| Automation (자동화) | 미출시 |

DB 이벤트에 반응하는 Automation 기능은 아직 공개되지 않았습니다. 현재로서는 에이전트를 통한 수동 호출만 가능합니다.

## 크레딧 소모

에이전트를 통해 워커를 호출하면 노션 AI 크레딧이 소모됩니다. 단순히 입출력 2회가 아니라, 에이전트의 내부 추론 과정(도구 선택, 파라미터 생성, 결과 해석 등)에서 **1회 호출당 약 5~6크레딧**이 소모됩니다.

# 워커스는 누구를 위한 것인가

노션 워커스는 "에이전트에게 자연어로 시키면 코드가 실행된다"는 경험을 제공합니다. 코딩을 할 줄 아는 사용자에게는 노션의 가능성을 크게 확장하는 도구입니다.

다만 알파 상태의 불안정성, 플랫폼 제약, 크레딧 소모를 감안하면, 지금 당장 프로덕션에 투입하기보다는 **구조를 이해하고 프로토타이핑하는 용도**로 접근하는 것이 적절합니다.
