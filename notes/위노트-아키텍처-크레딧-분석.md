# 위노트(WeeNote) 아키텍처·크레딧 구조 분석

> 작성일: 2026-05-18
> 목적: 위노트 경쟁 분석 및 Weestory 전략 포지셔닝 정리. 대화 내용 기록 (추후 PDF 변환용).

---

## 1. 출발점이 된 의문

위노트는 학교 PC에서 **로컬 LLM**(카카오 Kanana)을 돌리는 것을 핵심 셀링포인트로 내세운다.
그런데 가격표를 보면 **크레딧 차등 4단계**(프리 20 → 베이직 240 → 프로 2,400 → 프리미엄 9,600)가 있다.

로컬 모델은 한 번 깔면 한계비용이 사실상 0이다. 학교 PC가 추론 비용을 다 부담하므로
위노트 입장에서 사용자가 AI를 100번 쓰든 10,000번 쓰든 추가로 나가는 돈이 없다.
**그렇다면 크레딧을 단계별로 쪼개 팔 이유가 없다.** — 이게 풀리지 않던 의문이었다.

---

## 2. 퍼즐의 답: 위노트는 "로컬 전용"이 아니라 하이브리드

코드 분석(`app.asar` / `out/main/index.js` / 렌더러 번들 / `docs/ai-config-api-spec.md`)으로
확인된 사실은, 위노트가 **로컬 + 클라우드 하이브리드 구조**라는 것이다.

### 2-1. 로컬 추론 계층 (확인된 사실)
- 추론 엔진: **KoboldCpp v1.102.3**, `127.0.0.1`에서 구동
- 모델: **카카오 Kanana 1.5** — 2.1B Instruct(Q5_K_M, 1.74GB), 8B Instruct(Q4_K_M, 5.1GB), GGUF
- CPU 전용 모드 지원, 4GB RAM 노후 학교 PC까지 대응
- 상담 기록은 전량 **온디바이스 처리** — 이 계층만 보면 외부 전송 없음
- 구모델(HyperCLOVA X SEED, EXAONE 3.5)은 폐기, Kanana로 일원화
- UI상 명칭: **AI / AI Pro** (로컬 실행)

### 2-2. 클라우드 추론 계층 (확인된 사실)
- 모델 명칭: **"WeeNote AI Cloud (Beta)"**, `isCloud` 플래그, 인터넷 연결 필수
- 실제 모델: **Google Gemini 2.5 Flash**(RAG 질의용) + **Gemini 2.5 Flash-Lite**(일반 추론)
- Gemini API 키는 라이선스 서버 `/api/ai/config` 엔드포인트에서 기기 인증 후 수신
- File Search Store 기반 RAG: `weenote-rme253kmkttn`(폭력), `weenote-wee-5tv0km3mbi75`(Wee 주제)
- 클라우드 모델 선택 시 KoboldCpp 로컬 다운로드/설치 체크를 **전부 우회**
- UI상 명칭: **AI Max / AI Cloud** (외부 Google 서버 처리)

### 2-3. 중앙 설정 관리 (확인된 사실)
- 앱 시작 시 라이선스 시리얼(Bearer) + 기기 ID로 `/api/ai/config` 호출
- 서버가 **Gemini API 키 + 계층별 시스템 프롬프트**를 동적 전달
- 앱 재배포 없이 프롬프트 개선·키 로테이션 가능, 서버 불통 시 하드코딩 프롬프트로 폴백
- 라이선스 단위로 어떤 AI 계층·키를 받을지 접근 제어

## 3. 크레딧 구조가 이해되는 지점

위노트의 가격 사다리(베이직 25,000원 → 프로 40,000원 → 프리미엄 80,000원)와
크레딧 차등은 **로컬 모델로는 설명되지 않고, 클라우드(Gemini) 계층을 전제해야만 설명된다.**

- 로컬 Kanana 추론은 한계비용 0 → 단계별 과금의 근거가 될 수 없음
- Gemini 2.5 API 호출은 **호출당 실제 비용이 발생** → 위노트가 매월 Google에 지불하는 변동비
- 따라서 가격 사다리와 크레딧 단계는 **클라우드 사용량을 미터링하기 위한 장치**로 보는 것이 합리적

가격표상 'AI · AI Pro 무제한'(로컬)은 전 등급 공통이고,
'AI Max'(클라우드)는 프로·프리미엄 등급에만 열린다. 즉 **돈을 더 낼수록 열리는 가치가
클라우드 계층 쪽에 몰려 있다.**

> ※ 크레딧 1단위가 정확히 클라우드 호출 몇 회에 대응하는지는 가격표만으로 단정할 수 없다.
> 다만 구조적 결론은 분명하다 — 순수 로컬 앱이라면 4단계 크레딧 사다리 자체가 필요 없다.
> 크레딧 차등의 존재가 곧 **유의미한 유료 클라우드 의존성의 방증**이다.

### 위노트 가격표 (2026-05 기준)

| 등급 | 가격(월) | 크레딧 | AI 계층 |
|---|---|---|---|
| 프리 | 무료(1개월 체험) | 20 (2,000원 상당) | AI·AI Pro 무제한 |
| 베이직 | 25,000원 (연 300,000원) | 240 (24,000원 상당) | AI·AI Pro 무제한 |
| 프로 | 40,000원 (연 480,000원) | 2,400 (24만원 상당) | **AI Max 무제한** (AI·AI Pro 포함) |
| 프리미엄 | 80,000원 (연 960,000원) | 9,600 (96만원 상당) | **AI Max 무제한** (AI·AI Pro 포함) |

→ '로컬 LLM'을 전면에 내세웠지만, 가격 차별화의 무게중심은 클라우드(Gemini) 계층에 있다.
'로컬 우선' 마케팅과 실제 과금 구조 사이에 간극이 존재한다.

## 4. 위노트 '동의' UX의 구조적 하자

클라우드 계층은 학생 상담 데이터를 Google 서버로 보낸다. 위노트는 이를 **클라우드 모델
활성화 시 경고 모달 + 클라이언트단 정규식 마스킹(전화번호·학년/반/번호 패턴)**으로 처리한다.
그러나 이 '동의' 설계에는 한국 개인정보보호법 관점에서 네 가지 구조적 하자가 있다.

### (1) 동의 주체가 틀렸다
모달에 동의를 클릭하는 것은 **교사**지만, 상담 기록의 정보주체는 **학생(미성년자)**이다.
미성년자 개인정보 처리에는 법정대리인(보호자) 동의가 필요하다. 교사 클릭만으로는
동의가 법적으로 유효하게 성립하지 않는다.

### (2) 익명화 주장이 불충분하다
전화번호·학번 정규식 마스킹만으로는 **재식별 위험을 차단하지 못한다.**
학교라는 유한 모집단 안에서 상담 서사(narrative) 내용 자체로 특정 학생을 추론할 수 있다.
"마스킹했으니 익명 데이터"라는 전제는 익명화 기준을 충족하지 못한다.

### (3) 권한 위임의 오류
제3자(Google)에게 민감한 학생 데이터를 전송할지 여부는 **기관 차원의 보안정책 결정**이다.
이를 개별 교사의 모달 클릭으로 위임하는 것은, 그런 판단을 내릴 권한·정보가 없는
개인에게 기관의 책임을 떠넘기는 구조다.

### (4) 국외 이전 고지 누락
Google API로의 전송은 **개인정보 국외 이전**에 해당한다. "저장·학습에 쓰지 않는다"는
일반 문구만으로는 부족하며, 국외 이전에 대한 별도의 구체적 고지·동의 절차가 필요하다.

### 정리
위노트는 본래 필요한 무거운 기관 절차 — **개인정보 영향평가(PIA), 위탁/처리 계약,
정보보안 심의** — 를 **동의 모달이라는 지름길로 우회**했다.
동의 계층이 '개별 교사 옵트인'에 머물러 있는 것이 핵심 하자다.

## 5. Weestory 2안의 위치 — 한계와 진짜 강점

내(사용자) 2안은 **마스킹 기반** 접근이다. 위 (1)(2)번 하자, 즉
**동의 주체 문제와 마스킹만으로 익명화가 불충분하다는 한계는 2안도 동일하게 안는다.**
이 점은 솔직하게 인정한다.

그러나 2안에는 위노트가 갖지 못한 분명한 보안상 이점이 있다:

- **국외 서버로의 정보 이전이 없다.** → (4)번 하자에 해당하지 않음
- 추론이 **사측이 구축한 서버 내부에서만** 이뤄진다 → 처리 경계가 통제 가능한 국내 인프라 안에 머묾
- 제3자(Google) API에 민감정보가 닿지 않으므로, 국외 이전 고지·위탁 범위가 단순해진다

즉 2안의 마스킹 한계는 위노트와 공유하지만, **데이터가 어디서 처리되는가**라는 축에서는
2안이 구조적으로 우위다.

### 전략 포지셔닝 (B2G 차별화)
- 위노트가 우회한 지점 = Weestory의 차별화 기회
- 개별 동의 UX의 편의가 아니라 **기관 컴플라이언스 정합성**으로 경쟁한다
- 교육청 조달 절차에 정식 PIA·위탁계약·보안 심의를 태워 **기관 승인으로 데이터 처리를 정당화**
- 개별 교사 클릭이 아니라 기관 인가(institutional authorization)로 동의 계층을 끌어올린다

## 6. 한 줄 요약

> 위노트의 크레딧 차등은 "로컬 LLM"이 아니라 그 뒤에 숨은 **Gemini 클라우드 계층**의 변동비를
> 미터링하는 장치다. 그 클라우드 전송을 교사 1인의 동의 모달로 정당화한 것이 위노트의
> 구조적 약점이고, **국내 서버 내부 처리**가 Weestory 2안의 진짜 강점이다.
