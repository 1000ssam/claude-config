# 노션 템플릿 → 앱 전환 프롬프트

## 1단계: 노션 AI용 (PRD 작성)

```
이 템플릿의 페이지 콘텐츠와 DB 구성을 분석해서
앱 구현을 위한 PRD를 작성해줘.

출력 형식:
1. 앱 개요 (한 줄 목적 + 핵심 사용자)
2. 데이터 엔티티 표
   | DB명 | 속성명 | 속성 타입 | 앱 필드명(camelCase) | 비고 |
   Relation/Rollup/Formula는 비고에 명시
3. 핵심 기능 목록
   - 뷰/필터/관계 구조 + 페이지명/DB명 등 텍스트 맥락까지 종합해서 도출
   - 단순 CRUD는 제외
```

## 2단계: 클로드 코드용 (기술 설계)

```
## 입력: 노션 템플릿 PRD
[노션 AI 아웃풋 붙여넣기]

## 작업
위 PRD를 기반으로 기술 설계를 해줘.

1. 데이터 모델 설계
   - PRD의 데이터 엔티티 표를 그대로 구현
   - Relation 매핑
   - Rollup/Formula → computed 필드로 뺄지 DB 컬럼으로 저장할지 판단 후 주석으로 이유 명시

2. 서비스 레이어 설계
   - PRD의 사용자 스토리별로 필요한 서비스 함수 시그니처 목록
   - 노션 필터·수식에 해당하는 로직은 TypeScript 의사코드로 작성
```
