## Executive Summary

전국 약 9,300개 위클래스에는 공통된 문제가 있습니다.
상담 일지를 쓰고, 나이스(NEIS)에 다시 올리고, 관리자 보고서를 또 쓰는 **3중 행정 반복**입니다.

**위스토리**는 현직 교사가 직접 도메인 지식을 설계에 녹여 만든 상담교사 전용 기록 관리 앱입니다.
케이스 관리 → 일지 작성 → NEIS 업로드 서식 자동 출력 → 보고서 생성까지 하나의 흐름으로 연결합니다.

완성된 MVP, **1,100만원의 검증된 매출**, 그리고 **3.5만 팔로워 전문 상담교사 인플루언서**를 포함한 팀을 바탕으로 테크빌과의 공동 개발·유통 협업을 제안합니다.

<!-- Slide 3: 문제 ① — 3중 행정 반복 -->

## 문제 — 반복되는 3중 행정 부담

위클래스 상담교사는 한 건의 상담을 마치고 나서 다음의 작업을 **각각, 별도로** 수행합니다.

1. **상담 일지 작성** — 날짜, 회기, 주호소, 상담 내용, 개입 방법 등
2. **NEIS 업로드** — 상담구분 대분류/중분류/소분류 등 십수 개 카테고리를 다시 입력
3. **관리자 보고** — 요청이 올 때마다 별도 보고서 작성

동일한 정보를 서로 다른 시스템에 반복 입력하는 데 쓰이는 시간은,
학생과 마주 앉아 상담에 써야 할 시간입니다.

---

<!-- Slide 4: 문제 ② — 기존 도구의 한계 -->

## 문제 — 기존 도구의 한계

현재 상담교사들이 사용하는 도구는 다음과 같습니다.

| 도구 | 한계 |
|------|------|
| **엑셀 / 메모장** | 구조 없음, 검색 불가, 분실 위험 |
| **나이스 직접 입력** | 카테고리 입력 구조가 복잡하여 기록 용도로는 부적합 |
| **노션 템플릿** | 구조 강제 불가, 기본 지식 요구, 업데이트 미지원 |

각 도구가 제 역할을 못하면서 **데이터는 분산되고, 행정은 비효율적으로 반복**됩니다.

## 솔루션 — 위스토리 (Wee-story)

위스토리는 **위클래스 상담교사 전용 기록 관리 로컬 앱**입니다.

- **Electron 기반** — 학교 PC에 설치, 별도 서버 불필요
- **로컬 저장** — 민감한 상담 데이터가 외부로 전송되지 않음
- **하나의 흐름** — 케이스 등록 → 세션 기록 → NEIS 출력 → 보고서까지 원스톱

> 3중 행정 반복을 하나의 워크플로로 통합합니다.

## 핵심 기능 ① — 상담 기록 관리

### 내담자 · 케이스 · 세션 — 정규화된 데이터 구조

- **학생 DB**: 학적 이력 관리, 재학/졸업 상태 추적, NEIS 반배정 Excel 가져오기
- **케이스 관리**: 의뢰경로, 주호소, 사례 개념화를 케이스 단위로 체계적 관리
- **세션 기록**: 회기별 상담 내용, NEIS 카테고리(대분류/중분류/소분류) 자동 매핑
- **그룹 상담**: 하나의 케이스·세션에 다수 학생을 연결하는 다대다 구조 지원

> 같은 정보를 두 번 입력하지 않습니다.
> 한 번 기록하면 NEIS 서식과 보고서에 자동으로 반영됩니다.

<!-- Slide 10: 보안 설계 -->

## 보안 설계

상담 정보는 학생의 가장 민감한 개인정보입니다.
위스토리는 이를 전제로 보안을 설계했습니다.

| 계층 | 구현 |
|------|------|
| **저장** | 학교 PC 로컬 저장, 클라우드 전송 없음 |
| **암호화** | 학생 이름 등 민감 필드 개별 암호화 (field-level encryption) |
| **접근 통제** | 마스터 비밀번호 설정, 자동 잠금 화면 |
| **복구** | 비밀번호 분실 시 복구 코드 기반 재설정 |
| **DB 무결성** | CHECK 제약 조건으로 데이터 정합성 보장 |

> 개인정보보호법 및 학생 상담 정보 보안 요건을 충족합니다.

## AI 통합 — 함께 풀고 싶은 과제

상담 일지 자동 작성, 사례 개념화 보조, 위기 징후 알림 등
AI가 기여할 수 있는 영역은 명확합니다.

다만 상담 정보는 민감한 개인정보이기 때문에,
외부 AI API 활용 시 **기술적·법적 안전망 확보가 선행**되어야 합니다.

**검토가 필요한 선택지:**

- 로컬 LLM 적용
- 익명화 처리 후 API 전송
- 기타 기술적 대안

> 어떤 방식이 현실적으로 가능한지, 테크빌과 함께 검토하고 싶습니다.

## 시장성 검증 — 우리는 이미 팔아봤습니다

위스토리는 아이디어에서 출발한 게 아닙니다.
2024년, 전국 최초로 위클래스 전용 노션 템플릿(Wee Class Pro)을 출시하여 시장을 개척했습니다.

> **2024년 10월 ~ 2026년 4월 (약 1년 6개월)**

| 지표 | 수치 |
|------|------|
| 무료 버전 다운로드 | **1,954건** |
| 유료 판매 건수 | **219건** |
| 유료 전환율 | **약 11.2%** (일반적 프리미엄 전환율 2~5% 대비 2배 이상) |
| 누적 매출 | **약 1,100만원** |
| 별도 마케팅 예산 | **없음** |

이 수치는 두 가지를 증명합니다.

1. **수요는 실재한다** — 전용 도구에 돈을 내는 상담교사가 이미 존재합니다.
2. **개인의 한계가 명확하다** — 배포·마케팅·보안 인프라 없이는 성장에 천장이 있습니다.

<!-- Slide 13: 왜 앱인가 -->

## 왜 앱인가 — 노션 템플릿의 한계

노션 템플릿으로 시장을 개척했지만, 한계도 분명합니다.

| 노션 템플릿의 한계 | 위스토리 앱의 해결 |
|---|---|
| 사용자가 구조를 임의로 변경 → 데이터 일관성 붕괴 | 정규화된 DB 스키마로 구조 강제 |
| 노션 기본 지식 필요 → 진입 장벽 | 전용 UI로 학습 비용 제로 |
| 업데이트 자동 적용 불가 | 앱 업데이트로 일괄 배포 |
| 데이터 보안 통제 불가 (노션 서버 저장) | 로컬 저장 + field-level 암호화 |

> 동일한 도메인 지식, 동일한 데이터 스키마를
> **노션의 한계를 걷어내고 앱으로 구현**한 것이 위스토리입니다.

## 시장 포지셔닝

현재 시장에 알려진 전용 솔루션은 **위노트(WeeNote AI)** 단 하나입니다.

| | 위노트 | 위스토리 + 테크빌 |
|---|---|---|
| **개발 주체** | 1인 사업자 | 현직 교사 + **중견 에듀테크 기업** |
| **판매 채널** | 자체 홍보 | **쌤동네 생태계** + 3.5만 인플루언서 + B2G 영업망 |
| **가격 모델** | 개인 구독 (교사 사비 부담) | **기관 라이선스** (교육청 예산) |
| **고객 지원** | 1인 CS | **기업 수준 CS + 교사 연수** |
| **브랜드 신뢰** | 개인 법인 | **테크빌교육 브랜드** |

> 제품력은 시간이 해결합니다.
> 하지만 **신뢰, 채널, 브랜드**는 돈으로도 시간으로도 쉽게 만들 수 없습니다.

## 팀 소개

### 김용천 — 현직 교사, 바이브코더

- 노션 공식 앰버서더, 공식 추천 템플릿 크리에이터

- 테크빌교육과 다수의 협업 경험 보유
- **역할**: 현장 피드백 수집, 추가 기능 기획, 현장과 제품 사이의 연결고리

### 조열음 — 전문상담교사, 교육 인플루언서

- 현직 전문상담교사, 위클래스 운영 도메인 지식 보유
- Wee Class Pro 기획 단계부터 참여 — 실제 업무 흐름을 제품 설계에 반영
- **인스타그램 팔로워 3.5만명** — 상담교사 커뮤니티 내 최상위 인플루언서
- **역할**: 초기 확산의 중심, 사용자 커뮤니티 운영

> **현직 교사가 직접 설계하고, 현직 교사가 홍보합니다.**
> 이것이 일반 에듀테크 스타트업과 다른 본질적 차이입니다.

## 비즈니스 모델

### 타겟: B2G (교육청 / 학교)

개인 상담교사에게 구독료를 청구하는 방식(B2C)은 지속 가능하지 않습니다.
위스토리는 **교육청 및 학교 단위 일괄 라이선스** 계약을 목표로 합니다.

| 구분 | 내용 |
|------|------|
| 계약 단위 | 교육청(17개), 교육지원청(176개) |
| 가격 전략 | 라이트(AI 제외) 버전을 저가 책정하여 기관 도입 문턱 낮춤 |
| 확장 모델 | 라이트 → AI 프리미엄 업셀링 |

### 수익 배분

Revenue Share 방식을 기본으로 제안하며, 구체적인 배분 비율 및 조건은 협의를 통해 결정합니다.

## 협업 제안 — 역할 분담

### 제안자 측 (김용천 + 조열음)

- 완성된 MVP 제공 (케이스 관리 → 일지 → NEIS 출력 → 보고서)
- 위클래스 도메인 지식 및 설계 방향 제시
- 조열음 채널을 통한 초기 마케팅 및 커뮤니티 확산

### 테크빌/체더스 측 (요청 사항)

- 코드 리뷰 및 품질 고도화 (보안, 안정성, 배포)
- 교육부 에듀집 등록 등 공식 채널 등재
- 교육청 단위 B2G 영업 및 계약 체결
- 쌤동네를 활용한 상담교사 연수 기획·운영
- 마케팅 기획 및 집행

## 왜 지금인가 — 바이브코딩 시대의 새로운 협업 공식

2026년 현재, 교육 현장에 새로운 현상이 나타나고 있습니다.
도메인 지식을 가진 **현직 교사 바이브코더**들이 직접 앱을 만들기 시작했습니다.

**이들이 가진 것:**
수십 년간 축적된 현장 도메인 지식, 페인 포인트를 정확히 아는 눈, 교사 커뮤니티 내의 신뢰와 네트워크

**이들에게 없는 것:**
코드 품질 고도화 역량, 보안 인증 및 배포 인프라, 기관 대상 B2G 영업 채널

> **위스토리는 이 협업 공식의 첫 번째 프로토타입입니다.**
