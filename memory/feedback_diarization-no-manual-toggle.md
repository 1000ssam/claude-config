STT 화자 분리(diarization) 솔루션을 제안할 때 **"화자별로 사용자가 매번 버튼을 눌러 라벨링하는" UX 패턴은 절대 제안하지 않는다.**

구체적으로 금지되는 패턴:
- "선생님 발화 버튼" / "학생 발화 버튼" 두 개 마련해서 화자마다 토글
- 발언 직전에 누가 말할지 미리 선택하는 UI
- 전사 결과를 사후에 사람이 화자별로 끌어다 분류하는 UI

**Why**: 상담 같은 실시간 대화에서 사용자가 매 발화마다 버튼을 누르는 건 흐름을 깨고 집중을 빼앗는 끔찍한 UX. "구현 쉬워서" 제안하는 건 사용자에겐 비용 전가. 자동 분리가 안 되면 "안 된다"고 말해야지, 사용자에게 노가다를 떠넘기는 우회를 추천하면 안 된다.

**How to apply**:
- 화자 분리는 자동(pyannote, WhisperX, RTZR API, Sortformer 등) 솔루션 우선 제시
- 자동 솔루션이 비용·기술 제약으로 안 되면 "현 시점 불가" 또는 "후속 시나리오로 미룸"이라고 답함
- 정말 수동이 필요하면 **녹음 후 LLM 후처리로 자동 라벨 추정 → 사용자 검수**까지가 마지노선. 그 이상 사용자 손이 가는 건 금지
