running-challenge-3에서 **러닝 인증을 했는데 진행도 SMS가 안 오면**, 가장 먼저 의심할 것은 앱/발송 로직이 아니라 **노션 자동화(웹훅 트리거)가 우리 웹훅을 호출하지 않은 것**이다.

**진단 신호 (2026-05-30 4기 시작 직후 실제 발생)**:
- `발송 상태`=진행(열림)이고 인증 기록·챌린저→신청자→전화번호 관계·참여확정 전부 정상인데도 문자 안 옴.
- **결정적 단서: 발송내역 DB(`NOTION_DS_MESSAGE_LOG`)에 로그 0건.** 웹훅이 한 번이라도 호출되면 성공/실패 무관하게 `logMessage`로 로그가 남는다. 0건 = 웹훅이 아예 안 닿음. 러닝기록 `문자발송완료`도 false 그대로.

**검증 방법 (서버 정상 여부)**: `POST https://running-challenge-3.vercel.app/api/webhook/run-added` 에 헤더 `x-webhook-secret`(=Vercel env `WEBHOOK_SECRET`).
- 빈 페이로드 → `400 page id not found` = 배포·시크릿·핸들러 정상.
- 실제 러닝기록 page id(`{"id":"..."}`) → 진행도 SMS 즉시 발송 + 멱등 마킹. **누락 문자 수동 복구 수단**으로도 쓸 수 있음(멱등, 문자발송완료=false인 건만 발송).

**실제 원인 (2026-05-30 4기 사례, 확정)**: 자동화 설정은 "전부 멀쩡"했고 인증 추가 경로도 동일했는데, **세팅 직후 처음 2건(16:41/16:42 KST)은 자동화가 미발화 → 같은 방식의 3번째(16:51)는 정상 발화**. 즉 **노션 자동화는 새로 만들거나 DB 구조 변경 직후 초반 트리거를 조용히 흘리다가 곧 안정화되는 일시적 미발화**가 있다. 설정 오류로 단정 말 것 — 몇 분 뒤 재시도하면 살아날 수 있음. (자동화 화면을 한 번 열어 재저장하면 재활성화 효과도 있음.)

**진단 분기 (실전 누락 의심 시)**:
- 발송내역 로그 0건 → 웹훅 미발화 (노션 자동화 문제: 일시 미발화/미연결/일시정지/시크릿·페이로드로 401·400 후 로그 전 종료).
- 발송내역에 '실패' 로그 → Solapi 문제(잔액·발신번호 등).
- 401/400(로그 안 남음) vs 미발화 구분은 **Vercel 함수 로그**를 켜둔 채 테스트 인증 1건으로 판별. (단 Hobby 플랜은 `vercel logs <url>` 실시간 tail만, 과거 로그 없음.)

**알려진 잔존 리스크 (HANDOFF.md)**: Hobby 10초 함수 timeout — 동시 인증 burst가 크면 일부 웹훅이 죽어 산발적 누락 가능. 매일 08:00 KST 크론이 별개 경로로 일부 보완. 산발적·burst성 누락이면 Pro 업그레이드/노션 throttling 검토.

**복구 도구 = 멱등 수동 호출**: `문자발송완료=false`인 러닝기록 page id로 위 웹훅 POST → 누락분만 재발송 + 마킹. 진단(파이프라인 정상 여부)과 복구를 동시에 함.

게이팅 스위치(발송 상태)는 [[project_running-challenge-cron-toggle]] 참조(별개 원인).
