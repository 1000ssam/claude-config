콘텐츠 작성 시(블로그·SEO 포스트·DB 페이지 본문 등) 출처 자료에 명시되지 않은 기능·시나리오·플랜 제한·환경 제약·후기·인용·수치 등을 절대 발명하지 않는다.

**Why**: 2026-05-05 세션에서 노션톡 템플릿 포스트 21건 일괄 작성 시 서브에이전트들이 마켓 소개글에 없는 내용을 60+개 블록 분량 발명. 특히 한국 K12 교사에게 부적용인 "Notion for Education" 권장, 검증 안 된 "노션 무료 플랜 블록 수 제한" 단정, 마켓 글에 없는 "iOS 단축어" 시나리오 추가, "통근길·산책 중" 같은 후킹 카피 발명 등이 발견됨. 사용자 신뢰 즉시 손상 + K12 잘못 안내 같은 실질 피해 발생. 사용자 명시: "없는 걸 지어내지 않는 게 가장 중요함 명심해라."

**How to apply**:
- 출처(마켓 description / body_text / 사용자 컨펌 사실)에 없으면 본문에 적지 않는다
- "노션 일반 사실이라 적어도 되겠지", "정확히는 모르지만 보통 그럴 것" → 함정. 적지 않음
- 후킹용 감성 카피·시나리오·수치도 발명 대상에 포함 ("의지력보다 시스템", "30권 중 5권만 남는다" 등)
- 부정 단정도 발명 ("외부 도구 없이 작동", "AI 기능 미포함" 등 — 출처에 명시 없으면 적지 않음)
- 의심되면 삭제. 빈 줄이 발명 문장보다 낫다
- 가격(price)·날짜·플랜 제한 수치 등 NEXT_DATA·pack에 비어있으면 단정 금지
- 서브에이전트에게 작업 위임할 때도 ALLOWED_TEXT 정의를 명확히 박고, 의심 시 삭제하도록 지시
