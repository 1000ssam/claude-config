SEO 블로그 글은 AI 자동 생성보다 본인이 직접 쓰고 재검토하는 방향을 선호한다.

**Why**: 2026-05-04 AI 독서노트 LITE 템플릿 소개글 작성 중 검증됨. 두 가지 접근을 직접 비교했음:
1. 기계적 SEO 톤 → 키워드는 맞지만 "기계적이다"
2. 스레드 톤(1000쌤 voice) 적용 → 자연스러워졌지만 여전히 본인이 직접 쓰는 것에 못 미침

결론: "글은 직접 쓰는 게 최고"라고 결론 — 스킬에 톤 가이드를 박아두는 작업도 보류.

**How to apply**:
- `seo-blog-writer` 스킬은 **그대로 둔다** (수정 금지). 메타데이터·구조 도출용으로는 유효함
- 사용자가 SEO 글 요청 시: 메타데이터(타이틀/슬러그/메타 설명) + H1 구조 + 키워드 배치까지만 AI가 도출하고, **본문 작성은 사용자에게 넘긴다** ("뼈대 잡았으니 본문은 직접 쓰실래요?" 같은 식으로 제안)
- 노션 업로드용 페이지 생성도 본문 미작성 상태로 만들어 두고 사용자가 채우게 하는 게 자연스러움
- 본인이 톤 재정비를 부탁하는 경우에 한해 톤 적용 리라이트 수행
