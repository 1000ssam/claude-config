description: 모든 기능 작업은 피처 브랜치에서, 사용자 직접 검토 전까지 main 머지 절대 금지
type: feedback
---

모든 기능 작업은 피처 브랜치(feature/*, fix/* 등)에서 진행한다. 사용자가 직접 검토하고 명시적으로 "머지해"라고 할 때까지 main에 머지하지 않는다.

**Why:** 사용자 검토 없이 main에 직접 커밋하여 큰 불만 초래 (2026-04-08). "메인에 바로 머지해버려"는 특정 이미 완성된 브랜치에 대한 1회성 지시였는데, 이후 신규 작업까지 main에 직접 커밋한 것이 문제.

**How to apply:**
- 새 기능/수정 시작 → 항상 피처 브랜치 생성
- "구현해", "ㄱㄱ", "해줘" 등 = 피처 브랜치에서 작업
- main 머지 = 사용자가 명시적으로 "머지해", "main에 반영해" 등 직접 지시할 때만
- CLAUDE.md의 기존 규칙("브랜치 명시 없으면 test 브랜치 먼저")보다 더 엄격하게 적용
