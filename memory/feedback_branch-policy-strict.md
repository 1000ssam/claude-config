모든 기능 작업은 피처 브랜치(feature/*, fix/* 등)에서 진행한다. 사용자가 직접 검토하고 명시적으로 "머지해"라고 할 때까지 main에 머지하지 않는다.

**Why:** 사용자 검토 없이 main에 직접 커밋하여 큰 불만 초래 (2026-04-08). "메인에 바로 머지해버려"는 특정 이미 완성된 브랜치에 대한 1회성 지시였는데, 이후 신규 작업까지 main에 직접 커밋한 것이 문제. **재위반 (2026-06-12)**: claude-skills 리포에서 "스킬 수정 시 직접 커밋+푸시"라는 관행 메모를 'main 직행 면허'로 확대 해석해 hwpx-skill 변경을 main에 직접 커밋·푸시함. 그 메모의 본뜻은 "다른 곳에서 편집해 동기화하지 말고 ~/.claude/skills에서 바로 작업하라"(경로 문제)였지 브랜치 정책 면제가 아니었음.

**How to apply:**
- 새 기능/수정 시작 → 항상 피처 브랜치 생성
- "구현해", "ㄱㄱ", "해줘" 등 = 피처 브랜치에서 작업
- main 머지 = 사용자가 명시적으로 "머지해", "main에 반영해" 등 직접 지시할 때만
- CLAUDE.md의 기존 규칙("브랜치 명시 없으면 test 브랜치 먼저")보다 더 엄격하게 적용
- **모든 리포에 적용**: 프로젝트 리포뿐 아니라 claude-skills, config-repo 등 설정·스킬 리포도 예외 아님
- **규칙 충돌 시**: 관행성 메모(예: "직접 커밋+푸시")와 본 정책이 충돌하면 항상 더 엄격한 쪽(본 정책)을 따르고, 애매하면 사용자에게 물어볼 것
