사용자가 검증 결과로 **"통과"**, **"오케이 통과"**, 또는 동등한 완료 표현을 쓰면 되묻지 말고 아래 흐름을 바로 실행한다:

1. 피처 브랜치에 의미 있는 단위 커밋 (여러 개여도 됨)
2. `git checkout main && git merge --no-ff <branch>`
3. `git push origin main`

**Why:** 사용자가 명시적으로 "내가 통과라고 하면 커밋하고 메인에 푸쉬해버려"라고 지시함 (2026-04-25 wee-log 작업 중). 매번 머지 승인을 받는 게 불필요한 왕복을 만든다. 검증 통과 = 머지 권한 부여로 취급.

**How to apply:**
- "통과", "오케이 통과", "OK 통과", "다 됐다 푸시해" 류의 명시적 완료 신호에만 적용.
- "잘 되네", "좋다", "완벽하다" 등 단순 반응은 기본 규칙(main 머지 금지) 유지.
- 여전히 브랜치 정책을 지킨다: main에 직접 커밋하지 말고 피처 브랜치 → 머지 경로를 쓴다.
- 푸시 후 한 줄 요약(브랜치, 머지 커밋 SHA, origin/main 상태)을 사용자에게 보고한다.
