# 인스타 캐러셀 스타일 가이드

## 워크플로

Threads 글 → 인스타 캐러셀로 재작업

## 전체 구조 공식

```
[커버] 후킹 헤드라인 + 1000쌤 @iooo_tttt
[도입] 공감/문제 제기 (1장)
[본문] 핵심 포인트 3~5장 (슬라이드당 1개)
[클로징] 정리 or 격려 + CTA + 1000쌤 @iooo_tttt
```

- 평균 6~8페이지
- 비율: 4:5 (400x500)

## 커버 헤드라인 유형

| 유형 | 패턴 | 예시 |
|------|------|------|
| 질문형 | "왜 ~해야 해?", "~뭐야?" | "노션에서 왜 Markdown 써야 해?" |
| 도발형 | "제발 ~하지 마세요", "~안 되는 사람!" | "노션 DB 태그 제발 이렇게 관리하지 마세요" |
| 숫자+혜택형 | "N가지", "N분이면 끝", "$N 아끼는 법" | "노션 AI 기능 4가지" |
| 웨비나형 | 날짜+시간+ZOOM+주제 | "1월 10일 20:00-22:00 ZOOM" |

- 헤드라인 1~2줄, 최대한 짧게
- 커버에 반드시 `1000쌤 @iooo_tttt` 브랜딩

## 바디 슬라이드 규칙

- 슬라이드당 포인트 1개만 (절대 욱여넣지 않음)
- 소제목 한 줄 (짧고 굵게) + 설명 2~4줄
- 전개 방식:
  - **번호 나열**: 1→2→3→4 스텝
  - **문제→해결**: 불편함 → "이렇게 하면 됩니다"
  - **비교 대조**: vs, 숫자 비교
  - **질문→답변**: 바로 답변

## 톤앤매너

- 구어체, 반말+존댓말 혼용 ("~해보세요", "~거든요", "~인 거죠")
- 친근하지만 정보 밀도 높음
- 이모지 거의 없음, 텍스트로 승부
- 짧은 문장 위주

## 클로징 공식

```
격려 한 줄 + 팔로우/저장 유도
1000쌤 @iooo_tttt
```

# 대표 샘플

## ★ 샘플 1: 클로드코드/마크다운 (조회수 1위 정보성 글)

유형: 질문형 커버 + 번호나열 + 트렌드 연결

```
[커버] 노션에서 왜 Markdown 써야 해?
       1000@iooo_tttt

[도입] 마크다운 몰라?!
       노션에서 왜 써야 하냐고? 알면 인생이 바뀜!
       # space bar → 제목
       1[] space bar → 체크박스
       - space bar → 목록
       이게 바로 Markdown

[본문1] 뭐가 좋은데?
        마우스 안 써도 됨. 키보드만으로 빠르게 작성.

[본문2] 진짜 이유?
        HWP 쓰던 시대는 끝남.
        AI 시대에, 마크다운은 필수

[본문3] Markdown → AI
        AI에게 명령하는 언어가 마크다운!
        프롬프트, AI 스킬 전부 .md 파일.

[본문4] 마크다운 = AI 시대의 문해력
        노션에서 AI 쓰려면 마크다운 Skills .md 부터.

[클로징] 마크다운 어렵지 않아요.
         시작이 반이에요.
         AI 시대의 문해력 :)
         노션 = 마크다운!
         화이팅!
         1000@iooo_tttt
```

## 샘플 2: 도발형 (DB 태그 관리)

유형: 도발형 커버 + 문제→해결 + 비교 대조

```
[커버] 노션 DB 태그 제발 이렇게 관리하지 마세요
       1000@iooo_tttt

[도입] 혹시 이러고 있진 않죠?
       태그 10개 넘으면 이미 위험 신호!

[본문1] 지금 당장 점검해보세요
        태그 안 쓰는 거 정리!
        9개 이하로 줄이는 게 목표!

[본문2] 태그 vs 관계형
        태그 20개 쓸 바에 관계형 DB로!
        태그: 단순 분류 30초
        관계형: 구조화된 연결 5분

[본문3] 핵심은 관계형
        4가지 상황별 정리법!

[본문4] AI 활용
        AI로 태그 자동 정리 가능
        AI가 추천하는 태그 구조

[클로징] 노션 DB 정리하세요!
         태그 다이어트 = 생산성!
         1000@iooo_tttt
```

## 샘플 3: 번호나열형 (노션 AI 기능 4가지)

유형: 숫자형 커버 + 깔끔한 넘버링

```
[커버] 노션 AI 기능 4가지
       1000@iooo_tttt

[도입] 노션 AI 써봤어?
       아직 노션 AI 안 쓰는 사람.
       노션 AI 없으면 손해!
       4가지만 알면 충분!

[본문1] 1번 AI 글쓰기
        AI로 초안을 뚝딱

[본문2] 2번 요약
        Database DB에서 @으로 요약!

[본문3] 3번 번역 (Esc)
        블록 선택하고 번역 끝

[본문4] 4번 자동화
        반복 작업을 자동으로

[클로징] 노션 AI 써보세요! 화이팅!
         꿀팁이었다면 저장!
         1000@iooo_tttt
```

## 샘플 4: 문제→해결형 (PARA 정리법)

유형: 도발형 커버 + 공감→프레임워크 전개

```
[커버] 노션 정리 안 되는 사람!
       1000@iooo_tttt

[도입] 정리가 안 돼!
       노션 쓰는데 어디에 뭘 넣어야 할지..
       페이지가 점점 늘어나기만 하고..
       P.A.R.A. 쓰면 됩니다!

[본문1] PARA 4가지
        P - 프로젝트 / A - 영역 / R - 리소스 / A - 아카이브
        PARA는 모든 정보를 이 4개로 분류.
        1가지 규칙만 기억하면 됨.

[본문2] P.A.R.A.란?
        "Area? Resource?" "Project?"
        구분이 어렵다면 '기한'으로 판단.

[본문3] 실전 적용
        - 기한이 있는 업무 → Project
        - 꾸준히 관리하는 영역 → Area
        - 참고 자료 → Resource

[본문4] 핵심 팁
        폴더 구조가 아니라 DB로!
        UI/UX까지 고려한 대시보드.

[클로징] 노션 정리는 구조가 전부!
         PARA + 관계형 + 대시보드!
         1000@iooo_tttt
```
