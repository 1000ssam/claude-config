wee-log "기록물 보관·이관" 패키지 기능은 **전국 공통 플로우만 커버**하기로 결정(2026-06-19). 그 이상(서울식 보유목록관리시스템, 지역별 차이)은 **오픈 후 현장 피드백 받아 확장**.

**Why:** 전국 17개 시도교육청 매뉴얼을 조사하니, 앱이 근거 삼은 서울 안내서의 "보유목록관리시스템 엑셀 업로드"가 서울 특유였음. 전부 맞추려면 끝이 없고 출시가 막힘.

**전국 공통 플로우(공공기록물관리법 기반, 검증됨)**: 상담교사(처리과)가 ①K-에듀파인 비전자문서등록 → ②색인목록·철표지 편철(보존상자) → ③이관목록(시행규칙 별지 제2호 서식) 작성 → ④인계인수서와 함께 실물 이관(→학교 문서고/교육지원청 기록관). **USB 단독 인계로는 완결 안 됨**(전국 만장일치). 상담일지 보존 10년.

**서울 특화(전국 공통 아님)**: "보유목록관리시스템"이라는 별도 시스템/엑셀 업로드는 서울 안내서에만 등장. 국가 법령·국가기록원·교육부 Wee 가이드·타 시도엔 없음. 전국 공통 등록 통로는 K-에듀파인(학교 자체 시스템).

**How to apply:** 패키지=1학년도×1기관 전용폴더(정본PDF+MANIFEST+이관목록 그 철 스코프), 잠긴파일(EBUSY) 처리, 보유목록(전체)은 패키지에서 분리해 별도 버튼/서울 옵션으로. `transfer-list.ts` 컬럼은 안내서 양식과 일치(별지 제2호 정밀대조=현장피드백 항목). 새 피처 브랜치에서 진행.

조사 매뉴얼 10종(중복 제거): `/mnt/c/dev/notes/wee-archival-manuals/` (INDEX.md에 검증등급·출처). 미확보 지역: 대구·울산·광주·전북·제주(공통플로우 따를 것으로 추정만).

**2026-06-20 진행:** 이관 패키지 철별 독립 폴더 재구성 + 보유목록 분리 + 잠긴파일 처리 = **main 머지·푸시 완료(0b4ca46)**. 🎯 **다음 작업=기록물 '상태' 모델 재설계 확정**: 수동 4단계 드롭다운 → [이관완료 수동 1클릭 체크 + 패키지상태 자동(미생성/최신/원본변경)]. 드리프트 기준점을 이관완료→패키지 내보내기로 이동(재내보내기=자동 해소). 편철=종이 물리 묶기(법18조 등록·분류·편철 별개), 에듀파인 업로드 아님. 구현 계획·진입점 전부 → `/mnt/c/dev/wee-log/HANDOFF-archival-status-model.md`.
