원고(에세이 등) 검토 메모를 달 때는 **아주 짧게**, 짚을 곳 + 가벼운 플래그("확인 필요"/"검토") 정도로만.

- 금지: "독자 혼란" 같은 강한 평가어, "반드시"·"~할 것" 같은 단정/명령형, 분류 태그 라벨.
- 예시: ❌ "모둠 수·과제가 안 맞아 독자 혼란. 앞뒤를 맞출 것" → ✅ "다섯 번째 모둠까지 등장. 확인 필요"

**Why:** 사용자(교사)가 직접 고쳐 쓸 거라 메모는 위치를 짚어주는 역할이면 충분. 과한 워딩은 부담스럽고 AI 티가 남.

**How to apply:** 사실 오류도 "통설은 X. 확인 필요" 식으로 판단 여지를 남겨 소프트하게. [[feedback_dont-overtrim-copy]] [[feedback_analysis-stay-descriptive]]와 같은 결.

**범위(중요):** 글이 **시기·조건을 특정하지 않았는데** 내가 임의 기준을 들이대거나, **원작(책 등)을 요약·인용하는 맥락**의 서술에 독립적 사실 잣대를 대는 건 과한 코멘트. 저자 본인 주장으로 쓴 사실 서술에만 점검. (이동욱1: 책 인용 단락의 수용소 서술에 "초기 수용소는 정치범 중심" 지적 → 과함, 삭제)
