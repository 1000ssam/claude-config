---
name: feedback_never-overwrite-existing-data
description: 테스트 시 기존 데이터가 있는 페이지/레코드에 절대 덮어쓰지 말 것
type: feedback
---

기존 데이터가 있는 페이지나 레코드에서 테스트하지 않는다. 반드시 신규 생성하거나 별도 테스트 페이지를 만들어서 테스트한다.

**Why:** 사용자가 수동으로 후보정 작업한 Notion 페이지를 테스트 목적으로 덮어써서 데이터를 날린 사고 발생 (2026-04-19).

**How to apply:** API 테스트, 스크립트 테스트 등 파괴적 작업은 항상 새 리소스를 생성해서 수행. 기존 리소스는 읽기 전용으로만 접근.
